    General information

    SYR devices can be operated either via the AccessPoint mode, as a WLAN client or over LAN.

    Commands:

    The SYR Connect products are addressed via HTTP requests. In the requests, between the following types are differentiated:

    • GET-Request: Read value
    • SET-Request: Write value

    Syntax / Structure of the URL:

    The command syntax looks like this for both requests:

    • http://-IP-:5333/ Device name/get/command
    • http://-IP-:5333/Device name/set/command/value

    Device names:

    Product Device name
    TrioDFR LS trio
    SafeTech + trio

    Answer:

    The answers to the requests are in JSON format. The answer includes the original Inquiry, as well as the result.

    Ports:

    The requests are made via port 5333.

    Error codes:

    Errors are not returned as status code but in the payload. The following errors can be returned:

    • The command does not exist
    • setPRF9:“MIMA – value outside the setting range

    Communication / control

    The control of SYR Connect products can be controlled via the SYR Connect app, the access point of Products or via the local network via WLAN/LAN.

    Local communication via the access point

    Switching on the access point is used in the respective instructions for use of the product described. The SSID of the access point consists of “SYR serial number”. A password is not necessary. The following IP address must be used in Accesspoint mode:

    • http://192.168.4.1

    Local communications in the network

    The LAN connection can be established into the device via plug-in of the network plug. No further actions are necessary for this.

    The WLAN connection can be menu-guided via the SYR app as well as via the command chain in the Accesspoint mode. More in the section “Network/Commands”.

    In the local network, the IP address of the selected interface of the device must be used for the placeholder as follows:

    • http://-IP:5333/

    Device information - Supported devices

    TrioDFR LS

    The SYR TRIO DFR LS Connect is a combination of a semi-automatic, rewindable Filter, a pressure reducer and a leakage protection. It is mounted directly behind the water meter on a SYR universal flange.

    • Prevents pitting and corrosion
    • Reliably regulates the water pressure
    • Reliably protects against expensive water damage and waste
    • Leakage protection self-terel phase can be activated
    • Up to 8 different profiles adjustable
    • Regular execution of a microclastic test
    • Manual emergency function
    • Emergency operation by battery in case of power failure

    SafeTech+

    The SafeTech+ Connect leakage protection is directly behind the water meter on the SYR Universal flange (DN20-32) mounted. With already existing universal flange, it can be used between this and a drinking water filter from SYR.

    • Reliably protects against expensive water damage and waste
    • Determines the hardness range, measures water pressure and temperature
    • Leakage protection self-terel phase can be activated
    • Up to 8 different profiles adjustable
    • Regular execution of a microclastic test
    • Manual emergency function
    • Emergency operation by battery in case of power failure

    Command Type Description Value range GET SET
    AB Bool Locking opening/closing true/false ✓ ✓

    To open or close the shut-off, the value of AB is set to true or false.

    • FALSE shut-off open
    • TRUE shut-off closed

    The current position of the shut-off can be retrieved with VLV.

    • 10 closure closed
    • 11 Stops closes
    • 20 Stop open
    • 21 Stops opened

    Microleak test

    Example MicroLacking Test Start

    GET /set/dex/true

{
  "setDEX":true
}

    Example Time of the micro leakage at 04:00 setting

    GET /set/dtt/04:00

{
  "setDTT": "04:00"
}

    The microcagon test can be started manually and terminated in the interval.

    During the test, the barrier is closed. The test takes approx. 4 minutes in demand. We recommend that you take the test in times with low water extraction.

    Validity

    TrioDFR LS SafeTech +
    ✓ ✓
    Version >v134 >v134

    Commands

    Command Type Description Value range GET SET
    DEX Bool Starts the MicroLeckage Test true X ✓
    DRP int Set the test interval 1-3 ✓ ✓
    DSV int Status of the microclastage. Checks the above test is Active or not. 0-3 ✓ X
    DTT string Sets the time when the test is performed. Format "00:00" - - ✓ ✓
    Attention during the test is the closure. Water withdrawal during the test will lead to a negative result.

    The command DSV retrieves the status of the micro leak.

    • 0 Test not active
    • 1 test active
    • 2 Broken due to pressure drop
    • 3 test skipped

    If the test is not started manually, but through the test interval, the device checks whether water was taken 180min beforehand. If water was taken in this period, the status (DSV) is set to the value 3 and the test is only executed again at the next time.

    For devices with pressure sensor, the test is stopped and the shut-off is reopened as soon as water is removed. The status is set to 2.

    The test interval can be set via DRP.

    • 1 The test starts every day
    • 2 The test starts weekly
    • 3 The test starts monthly

    The time can be set via DTT. The time is passed as a string in the format "HH:MM".

    The time is always set as UTC0. A time difference must be taken into account.

    Example

    • DTT Ã 04:00
    • DRP 1

    The micro leak test is performed daily at 04:00.

    The test is deactivated in the profiles in which you set to false.

    Commands

    Command Type Description Value range GET SET
    WFC string WLAN SSID true ✓ ✓
    WFD SSID and Key Delete ✓ ✓
    WFK string WiFi Key X ✓
    WFL json Provides a lister of available networks - - ✓ X
    WFR int WLAN signal strength in % 1-100 ✓ X

    WGW string WLAN IP ✓ X
    WIP string WLAN Gateway ✓ X
    EGW string Ethernet IP ✓ X
    EIP string Ethernet Gateway ✓ X
    MAC1 string MAC address WLAN interface ✓ X
    MAC2 string MAC-Address LAN Interface ✓ X
    The EGW and EIP commands are available only for LAN devices.

    The connection to a WLAN network can be established via the WFK and WFC commands.

    • The WFK command sets the WLAN key of the network

    • The WFC command sets the SSID

    After the WFC has been set, the connection attempt is started.

    WFK must always be set before WFC, otherwise the connection fails.

    The WFS command can be used to read the connection status of the WLAN interface.

    • 0 non-connected
    • 1 Connection is established
    • 2 composites

    The WFD command is used to delete the connection data from the device. WFD does not pass a value set/WFD.

    The devices inform the user about news, warnings, alarms and malfunctions via push message in the app or by e-mail.

    A distinction is made between alarms, warnings and notifications.

    • Alarms can be triggered due to both leakage and system-critical errors. Alarms always lead to the deactivation of the system.
    • Warnings are triggered when events occur which endanger smooth operation. For example, if the batteries have reached the end of their life.
    • Notifications are used to inform the user, a new update is provided or the filter backwash is overdue, so the user is informed.

    The message memory of the devices can also be retrieved via command. The message at the top of the output array always represents the most up-to-date message.

    Firewall configuration

    In order for your SYR devices to successfully communicate with SYR Azure services and download firmware updates, the following domains, IP address ranges and ports must be shared in your firewall.

    Azure IoT Hub Communication

    Enables centralized communication of devices with Azure IoT Hub.

    • FQDN: *.azure-devices.net
    • Ports:
      • 443(MQTT via WebSocket / HTTPS)
      • 8883(MQTT via TLS)
      • 5671(AMQP via TLS)

    IP address ranges: - Use the service tag AzureIoTHubin the following file: - Azure IP Ranges and Service Tags – Public Cloud

    Azure Blob Storage (Firmware Download)

    • FQDN: *.blob.core.windows.net
    • Port: 443, 80(Port 80 as backup)

    IP address ranges: - Service day: Storage

    Azure App Service

    • FQDN: *.azurewebsites.net
    • Port: 443

    IP address ranges: - Service day: AppService

    Time synchronisation (optional)

    • FQDN / IP: pool.ntp.org
    • Port: 123/UDP

    Recommended approach

    Whitelist after FQDNs (recommended)

    If your firewall supports FQDN-based rules, use domain names only.

    Alternative: IP addresses

    Use the regularly updated file: - Azure IP Ranges and Service Tags – Public Cloud

    Blob storage *.blob.core.windows.net 443
